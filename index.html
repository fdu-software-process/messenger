1. 简介

本项目是一个基于 TCP Socket 通信的聊天室软件，分服务器端和客户端开发，都提供了跨平台二进制程序。下图 1 是客户端程序运行截图。

项目功能特点如下：

  • 跨平台，提供 Windows、Linux、OS X 三种平台程序；
  • 简单设计的消息协议，在 TCP Socket 的基础上设计了类似 HTTP/1.1 的 Request-Response 以及 HTTP/2 的 Server Push（服务端推送）协议；
  • 消息协议包括文本信息、图片信息和表情（emoji），多人消息同步，整个客户端有简单的的断开重连和 IP 端口配置；
  • 简单的 UI（Webkit based）、响应式窗体和动画设计。

2. 通信协议设计

所有消息都是一个 JSON 字符串，传输于 server 和 client 之间。结构如下： 

{
  type,
  token*,
  timestamp,
  content
}

2.1. type 域

type 域是一个字符串，取值为 "push"、"res"、"login"、"msg" 之中一个。

  • 所有 server 发往 client 的消息，type 值为 "push" 或 "res"；
  • 所有 client 发往 server 的消息，type 值为 "login" 或 "msg"；

2.1.1. push

表示服务器端主动往客户端发送消息，用于向用户推送聊天室的新消息。

2.1.2. res

表示服务器响应客户端的请求。当 type 域取值为 "res" 时，消息需带有 token 域。

2.1.3. login

表示客户端请求登陆聊天室。当 type 域取值为 "login" 时，消息需带有 token 域。

login 消息的 content 域由用户昵称、服务器 IP 端口设置组成。

2.1.4. msg

表示客户端发送聊天室消息。当 type 域取值为 "msg" 时，消息需带有 token 域。

msg 消息的 content 由消息文本、发送时间组成。

2.2. token 域（可选）

token 域是一个随机生成的数字，用于匹配客户端的请求和服务端的应答。

在客户端发起请求消息时（login、msg），会随机生成一个 token 值。服务器收到此请求后做出的回应（res）带有同样的 token 值。客户端收到服务器的回应后，即可根据同样的 token 值，匹配一组请求和回应。

在服务端和客户端中，token 的生成和匹配都封装在了协议的 Request-Response 功能中。直接暴露给逻辑开发部分类似一个 HTTP POST 请求，API 接口与 jQuery 的 ajax 非常相似1。

2.3. timestamp 域

timestamp 域是一个数字，表示发送此消息请求的时刻，为标准的 POSIX time 格式2。

2.4. content 域

content 域是一个 JSON 文档，存储消息具体的信息。

3. 传输处理

为了发送长文本消息、方便切分被合并的 Socket 请求，消息发送前会做如下的分段处理（服务端和客户端同样）：

例如一个如下的消息，服务器需要主动推送消息给客户端： 

{
    type: "push",
    timestamp: 1449345271889,
    content:
        {
            type: "msg",
            data: 
                {
                    data: "It works.",
                    name: "ds",
                    time: 1449345271886
                }
        }
}

3.1. 格式化

把 JSON 转换成字符串，格式化、去掉多余字符： 

{"type":"push","timestamp":1449345271889,"content":{"type":"msg","data":
{"data":"It works.","name":"ds","time":1449345271886}}}

3.2. 切分

切分成段： {#segments} 

{"type":"push","timestamp":1449
345271889,"content":{"type":"ms
g","data":{"data":"It works.","
name":"ds","time":1449345271886
}}}

3.3. 编码处理

escape 掉 "." 符号，转换编码为 UTF-8 

{"type":"push","timestamp":1449
345271889,"content":{"type":"ms
g","data":{"data":"It works(dot)","
name":"ds","time":1449345271886
}}}

3.4. 加入段信息

加入段头信息，格式为 "段序号/总段数" 

.1/5.{"type":"push","timestamp":1449
.2/5.345271889,"content":{"type":"ms
.3/5.g","data":{"data":"It works(dot)","
.4/5.name":"ds","time":1449345271886
.5/5.}}}

3.5. 发送

将每一段以二进制格式送入缓冲区，以 Socket 发送。

3.6. 接收

消息接收端则完全反过来处理上述过程。

4. 技术架构

整个项目两端均由 NodeJS 完成，基于系统底层 Socket 调用来通信。UI 窗体是一个由 Electron 打包的 webkit 应用，GUI 设计模式为类 MVC，采用 AngularJS 库完成。代码文件如下：

  • client/app/ 下存放客户端的 GUI 文件，包括样式、view 和 controller 逻辑；
  • client/app/1f303.png 等文件为默认表情图片。

5. 可执行程序源码安装、二进制文件

项目源码上传于 https://github.com/quietshu/socket-chat-room，Win32 x64 和 darwin x64 的可执行文件可在 https://github.com/quietshu/socket-chat-room/releases 直接下载运行。

5.1. 从源码安装流程

首先安装 NodeJS，然后安装依赖工具： 

$ npm i babel -g
$ npm i electron -g

编译并运行 server： 

$ cd server
$ npm install
$ npm start

编译并运行 client： 

$ cd client
$ npm install
$ npm run build
$ npm start

5.2. 打包成 app：

OS X 下，在 client 目录中 $ npm run pack 即可。

Windows 下，先将 package.json 中 pack 命令中的 darwin 平台改为 win32 平台，再执行上述命令即可。
